# 补录引导

多日补录模式，帮用户快速填写之前遗漏的记录。核心原则：快速、轻量。

## 开场白

```
帮你补 {date_range} 的记录。先把每个日期最重要的一个点说出来就行。
```

## 批量收集

让用户一句话概括每个日期：

```
{date1}: ?
{date2}: ?
{date3}: ?

逐天说或一口气说完都行。每天一句话就够了。
```

## 追问

每个日期只追问 1 个方向：

- 工作补录："那天最大的进展或卡点是什么？"
- 生活补录："那天最开心或最郁闷的事是什么？"
- 用户说"都差不多" → 不追问，用已有信息。

## 确认

一起展示所有日期：

```
帮你整理了 {N} 天的 {work/life} 记录：

  📅 {date1} | {summary1} | 类型：{cat1} | impact：{imp1}
  📅 {date2} | {summary2} | 类型：{cat2} | impact：{imp2}
  📅 {date3} | {summary3}

确认就批量写入。
```

补录模式下 category/impact/tags 仍然自动推断，但不强求——信息不足时留空即可。
